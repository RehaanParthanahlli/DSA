DAWN 1 — CORE STACK ADT + PARENTHESES
------------------------------------------------------------
Stack‑Universe Coverage:  
- LIFO behavior and push/pop/top usage  
- Using stack for matching parentheses/brackets  
- Using stack for simple string cleanup  
- Implementing stack or stack‑like behavior explicitly  

Problems (order matters):  
1. Implement Stack using Queues (LeetCode 225) – simulate stack using two queues.  
2. Implement Queue using Stacks (LeetCode 232) – complement to 225; clarify LIFO vs FIFO.  
3. Valid Parentheses (LeetCode 20) – classic bracket check, push opening, match closing.  
4. Minimum Remove to Make Valid Parentheses (LeetCode 1249) – use index stack to mark invalid.  
5. Backspace String Compare (LeetCode 844) – stack to simulate typing and backspaces.  
6. Simplify Path (LeetCode 71) – directory stack to normalize Unix path.  
7. Remove All Adjacent Duplicates in String (LeetCode 1047) – stack pattern on characters.  

DAWN 1 – RULES:  
- For 20, 1249, 71: write down your stack *invariant* before coding.  
- If stuck > 30 minutes: stop that problem, write what blocked you, and retry next day.  

Skill Outcome:  
- Confident basic stack usage  
- Instinct: "Nested / matching / undo = stack"  
- Comfortable simulating modifications via stack

DAWN 3 — MONOTONIC STACK PATTERN
------------------------------------------------------------
Stack‑Universe Coverage:  
- Monotonic increasing/decreasing stack  
- "Next greater/smaller element" pattern  
- Using stack for nearest larger/smaller index queries  
- Recognizing problems that reduce to "next greater/smaller"  

Problems (core sequence):  
1. Next Greater Element I (LeetCode 496) – stack on nums2; map to nums1.  
2. Next Greater Element II (LeetCode 503) – circular array + stack.  
3. Daily Temperatures (LeetCode 739) – classic monotonic decreasing stack.  
4. Next Greater Element to Right / Left (pattern) – choose GfG "Next Greater Element" + "Nearest Smaller Element".  
5. Stock Span Problem – GfG classic, monotonic stack for previous greater.  

Hard / Pattern mastery (do after above):  
6. Trapping Rain Water (LeetCode 42) – re‑do after learning monotonic stack, even if you know two‑pointer.  
7. Sum of Subarray Minimums (LeetCode 907) – strict monotonic stack for previous/next smaller.  

RULES – MONOTONIC STACK:  
- Before coding, write: increasing or decreasing? values or indices? next or previous?  
- Template:  
  - Loop i from 0 to n‑1  
  - While stack not empty and current violates monotonic property → pop & process  
  - Push current index  

Skill Outcome:  
- Instant recognition of "use monotonic stack here"  
- Able to write generic next‑greater / next‑smaller template from memory  
- Comfortable with stack‑based O(n) solutions for array problems
